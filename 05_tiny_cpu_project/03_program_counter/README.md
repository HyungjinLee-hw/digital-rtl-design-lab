# 03_program_counter





## Overview / Design Goal



This module implements the Program Counter (PC) for the Tiny CPU project.



The Program Counter stores the address of the instruction currently being fetched. On every rising clock edge, it can:



* Reset to a defined start address
* Increment to the next instruction address
* Hold its current value
* Load a jump or branch target address



This module is designed for the current Tiny CPU architecture, which uses word addressing. Therefore, the normal PC increment is `+1`.

\## PC Update Priority



The PC update logic follows this priority:



1. reset
2. pc_write_en and pc_load
3. pc_write_en and normal increment
4. hold current PC value





## Word Addressing



This Tiny CPU treats each instruction memory entry as one word.



PC = 0 → instruction memory address 0

PC = 1 → instruction memory address 1

PC = 2 → instruction memory address 2





Therefore, normal execution increments the PC by 1.



A standard 32-bit RISC-V CPU usually uses byte addressing, so its PC commonly increments by 4 for each instruction. That behavior is intentionally not used in this Tiny CPU stage.





## PC Hold



When pc_write_en = 0, no assignment is made to the PC register during the clock edge.



pc_write_en = 0	// PC keeps its previous value





This will later be useful for CPU stalls, halt behavior, and memory wait states.





## Branch and Jump Support



The PC module does not calculate branch conditions or target addresses itself.



Those values will later be generated by the control unit and datapath.





## Testbench



The testbench verifies the following cases:



1. Reset sets PC to 0
2. Normal increment from 0 → 1 → 2 → 3
3. PC hold when pc_write_en = 0
4. pc_load is ignored when pc_write_en = 0
5. Jump or branch target load
6. Normal increment after a jump
7. Reset priority over jump target loading
8. 8-bit PC overflow and wrap-around



Example wrap-around behavior: 8'hFF + 1 → 8'h00





All tests passed in Vivado Behavioral Simulation.
